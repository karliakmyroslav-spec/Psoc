## 🛠 Хід виконання роботи

### 1. Підготовка апаратної частини проєкту

- За основу було взято проєкт з Лабораторної роботи №4. Зі схеми `TopDesign.cysch` видалено зайві компоненти (кнопка, RGB-світлодіоди, модуль UART).
- З каталогу компонентів (`Cypress -> Digital -> Functions`) додано апаратний таймер **Timer [v2.80]** та компонент переривання **Interrupt [v1.70]** (перейменований у `Timer_Int`).
- **Налаштування таймера:**
  - Resolution: `24-Bit` (UDB)
  - Period: `24000` (що відповідає перериванню кожну 1 мс при частоті тактування 24 МГц)
  - Run Mode: `Continuous`
  - Інші тригери та захоплення (Capture Mode) вимкнені.

### 2. Програмна реалізація динамічної індикації

- Написано обробник переривання `Timer_Int_Handler2`, який спрацьовує кожну мілісекунду.
- У перериванні реалізовано алгоритм динамічної індикації для 7-сегментного дисплея через зсувні регістри 74HC595:
  1. Спочатку дисплей повністю гаситься (захист від накладання символів/ghosting).
  2. У зсувний регістр відправляється код позиції (вибір розряду).
  3. Відправляється код самого символу (дані для сегментів).
  4. Здійснюється фіксація даних сигналом Latch.

## 🧑‍💻 Самостійне завдання: Кодовий замок та таймер

У рамках індивідуального завдання було створено повноцінну систему кодового замка з ефектами на 7-сегментному дисплеї:

1. **Введення пароля:** Введення здійснюється з матричної клавіатури. На дисплеї динамічно відображаються введені цифри (вирівнювання по лівому краю).
2. **Керуючі клавіші:**
   - Натискання `*` — скидає введений пароль (очищує буфер).
   - Натискання `#` — підтверджує введення та запускає перевірку еталонного пароля (у коді задано `7355608`).
3. **Успішна авторизація (`sequenceSuccess`):** \* На дисплеї по черзі з'являються слова: `ACCEPT`, `BOMB`, `HAS`, `BEEN`, `PLANTED`.
   - Після цього запускається таймер зворотного відліку, який виводить час у форматі `ХВ.СЕК` (наприклад, відлік 5 хвилин).
4. **Помилка авторизації (`sequenceFail`):** \* Виводиться повідомлення `FAILED`.
   - Реалізовано анімацію «біжучого рядка» (scrolling), що циклічно зсуває текст по дисплею.

### Фрагмент коду обробника динамічної індикації (переривання):

```c
CY_ISR(Timer_Int_Handler2)
{
    // Щоб уникнути "накладання" символів (ghosting), спочатку гасимо все
    FourDigit74HC595_sendData(0xFF);
    FourDigit74HC595_sendData(0xFF);
    Pin_Latch_Write(1);
    Pin_Latch_Write(0);

    // Виводимо поточний розряд
    FourDigit74HC595_sendData(0xFF & ~(1 << led_counter)); // Вибір позиції (анод/катод)
    FourDigit74HC595_sendData(display_buffer[led_counter]); // Дані символу (сегменти)

    Pin_Latch_Write(1);
    Pin_Latch_Write(0);

    led_counter++;
    if(led_counter > 7) led_counter = 0; // Перехід до наступного з 8 розрядів
}
```

## 📝 Висновок

Під час виконання лабораторної роботи було успішно засвоєно принципи роботи динамічної індикації, що дозволяє виводити інформацію на багаторозрядні дисплеї, використовуючи мінімум апаратних ресурсів мікроконтролера. На практиці налаштовано апаратний таймер та переривання для забезпечення стабільного оновлення дисплея без затримок у головному циклі програми. Крім того, реалізовано складну логіку обробки введеного пароля з матричної клавіатури, трансляції символів (включаючи літери) у коди 7-сегментного індикатора та розроблено алгоритми створення візуальних анімацій (біжучий рядок, таймер зворотного відліку).
